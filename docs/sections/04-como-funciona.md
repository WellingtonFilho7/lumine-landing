# Seção 04 — Como funciona o piloto do Lumine

## Propósito
Explicar o formato concreto do projeto piloto. O leitor precisa entender: quantas crianças, que idade, que frequência, onde fisicamente, o que vão estudar.

## Fundo
Warm creme (--bg-warm)

## Conteúdo esperado
- Turma pequena (definir número — 10-15 crianças?)
- Faixa etária do piloto
- Frequência (dias/semana)
- Local físico (sede, espaço cedido, etc.)
- Duração do piloto (semestre? ano?)
- O que será ensinado (visão geral, sem detalhar currículo)
- Março de 2026 como data de início

## Layout
- Container padrão
- Formato de timeline vertical ou steps numerados (não cards)
- Cada etapa/ponto com título curto + descrição breve
- Visual que sugira sequência e planejamento
- Alternativa: lista descritiva com termos em destaque

## Tom
Prático, concreto, transparente. Quem lê deve pensar "eles sabem o que vão fazer."

## O que NÃO fazer
- Não usar ícones decorativos genéricos
- Não fazer parecer maior do que é — é um piloto, não uma escola completa
- Não prometer resultados
